get与post的差异

  • 2019-05-14

今天给大家带来的是大部分程序员在面试中都会遇到的一个世纪难题 – get和post的不同点。本来这是一个几乎所有程序员都会准备到的面试题，但是往往他都是大部分程序员的噩梦。因为自己不知道这个题目的完整答案到底有多少，到底需要掌握多少才算是真正的掌握了get和post的区别。那么下面我会带来我个人对这个问题的一点总结与看法。

深入传输层的剖析get与post

首先get和post都是HTTTP的基本请求方式，但在请求方式上有着明显的区别。


上面是我在百度翻译上的截图，从他们各自的中文解释上不难看出他们表面的区别。

get就是请求得到数据，而post则是请求传输数据。

接下来我们对比一下get和post。

类别 get post
数据长度 因为get请求是将数据挂载到url上，所以get请求的数据长度受url的长度限制，（理论上url的长度都限制在2048个字符以内。） post请求方式的数据理论上是没有大小限制的，他的数据的大小是有服务器的设置决定的，可以理解为没有大小限制。
可见性 get请求方式是将数据暴露在url中，所以是可见的。 post请求方式没有在url中暴露。
安全性 因为get请求方式的数据是直接挂载到url上的，能够直接被用户看到，可以说是没有安全可言。 post请求方式的数据是存放在HTTP请求包的包体中，相对来说较为安全，但这仅仅是对于普通用户来说。
数据类型的限制 get请求只允许ASCII字符。 post请求对数据类型没有要求，可以接收任意的数据类型。
历史记录 get请求方式的数据会保留在历史记录里边。 post请求方式的数据不会保留在历史记录里边。
缓存 get请求的数据会被浏览器主动缓存。 post请求的数据不会被默认缓存，除非主动设置。
书签 get请求可以被收藏到书签。 post请求不能被收藏到书签。
刷新或者后退 get请求不会重新发起请求 post请求会被重新提交。

介绍一下HTTP协议。

HTTP协议是Hyper Text Transfer Protocol（超文本传输协议）的缩写,是用于从万维网（WWW:World Wide Web ）服务器传输超文本到本地浏览器的传送协议。HTTP定义了客户端与服务器交互的不同方法，其中最基本的四种方法分别是PUT、DELETE、POST、GET。他们分别对应着对资源的 增、删、改、查 四种操作。
get请求会将http header和data一同发送出去，服务器响应200后返回数据。
post请求首先将 http header发送出去，服务器响应100后继续发送data 然后服务器响应200后返回数据（一般都是是否成功）。、
这样一对比就会显示出来get的效率要比post高出来一倍以上。（毕竟要像服务器发送两次请求）

再介绍一下 HTTP请求中请求头 Header里的 Content-Type。

Content-Type一般有这三种。
application/x-www-form-urlencoded ：数据被编码为名称/值对。这是标准的编码格式。
multipart/form-data ： 数据被编码为一条消息，页上的每个控件对应消息中的一个部分。
text/plain ：数据以纯文本形式(text/json/xml/html)进行编码，其中不含任何控件或格式字符。postman软件里标的是RAW。

form表单的enctype属性为编码方式，常见的有两种：application/x-www-form-urlencoded、multipart/form-data。默认值是application/x-www-form-urlencoded。
当请求方式为get时浏览器会用application/x-www-form-urlencoded的编码方式把form数据转换成一个字符串，用 ？ 连接url将数据追加到url的后边。生成新的url。
当请求方式为post浏览器会将form数据分装到http body中，如果没有type=file的控件的话，浏览器会使用默认的编码方式application/x-www-form-urlencoded进行编码，如果有type=file控件的话，将会使用multipart/form-data的编码方式进行编码。
当action为post且Content-Type类型是multipart/form-data，浏览器会把整个表单以控件为单位分割，并为每个部分加上Content-Disposition(form-data或者file),Content-Type(默认为text/plain),name(控件name)等信息，并加上分割符(boundary)。
